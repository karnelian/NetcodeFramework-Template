---

## 확정 절차

### 1. 게임 특성 정의

| 항목 | 값 |
|------|---|
| 장르 | *(예: 로그라이크 액션 타워 디펜스)* |
| 뷰 타입 | *(Top-Down / Side-View / 3rd Person / Isometric)* |
| 타겟 플랫폼 | *(PC / Mobile / Console)* |
| 타겟 해상도 | *(1920x1080 / 2560x1440 / 동적)* |

### 2. 아트 스타일 결정

| 항목 | 값 |
|------|---|
| 스타일 | *(로우폴리 / 스타일라이즈드 / 리얼리스틱 / 픽셀아트 / 셀셰이딩)* |
| 색상 팔레트 | *(주요 HEX 3~5개)* |
| 톤 | *(밝음 / 어두움 / 파스텔 / 네온)* |
| 레퍼런스 게임 | *(2~3개)* |

### 3. 기술 기준

| 항목 | PC | Mobile |
|------|-----|--------|
| 텍스처 해상도 (최대) | 2048px | 1024px |
| 텍스처 해상도 (일반) | 1024px | 512px |
| 폴리곤 수 (캐릭터) | ~30K | ~10K |
| 폴리곤 수 (프롭) | ~5K | ~2K |
| 스프라이트 해상도 (2D) | 512px | 256px |

## 에셋 추천 규칙

1. 이 가이드의 스타일 기준을 반드시 참조
2. Unity Asset Store, itch.io, Sketchfab 등에서 후보 검색
3. 후보 리스트 제공 (포함 내용, 스타일 적합도)
4. 선택 후 위 테이블 업데이트
5. **핵심**: 기준 에셋 팩과 스타일 일관성 유지

## 리소스 확보 전략

### 3D 프로젝트
| 리소스 | 확보 방법 |
|--------|-----------|
| 캐릭터/몬스터 | Asset Store 팩 + AI 3D 생성 (`ai-3d-pipeline`) + Mixamo 애니메이션 |
| 배경/맵 | ProBuilder + 환경 팩 + 프로시저럴 생성 (`procedural-generation`) |
| 이펙트/파티클 | 코드 기반 생성 (`procedural-generation`) |
| 사운드 | `audio-strategy` 참조 |
| UI | `ui-workflow` + CLAUDE.md UI 계층 규칙 |

### 2D 대안
| 리소스 | 확보 방법 |
|--------|-----------|
| 캐릭터 일러스트 | AI 이미지 생성 (Midjourney, Stable Diffusion) |
| 2D 애니메이션 | Spine / DragonBones |
| 픽셀아트 | Aseprite로 직접 제작 |
| 타일맵 | AI 생성 + 수동 정리 |

## 머티리얼 네이밍

- 머티리얼: `M_{용도}_{이름}` (예: `M_Character_Player`, `M_Environment_Metal`)
- 텍스처: `T_{용도}_{이름}_{타입}` (예: `T_Character_Player_Albedo`, `T_Character_Player_Normal`)

## 에셋 평가 체크리스트

AI 생성 모델/에셋 채택 시:
- [ ] 아트 스타일 가이드와 일관성 있는가
- [ ] 폴리곤 수가 기술 기준 이내인가
- [ ] UV 맵이 깨끗한가 (텍스처 이음새)
- [ ] 리깅 품질 충분한가 (캐릭터)
- [ ] URP 셰이더와 호환되는가
